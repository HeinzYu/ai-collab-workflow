# S01 — Prompt Engineering Standard

> 对应 ChatGPT 方案 §8。提供完整的推荐/禁止Prompt模板库，针对 Apple Native App（SwiftUI）开发场景优化。

---

## 1. 核心原则

### 推荐的Prompt特征
- **具体**：明确指定文件、函数名、行数范围
- **有上下文**：引用现有代码或文档路径
- **单一目标**：每个Prompt只解决一个问题
- **可验证**：包含明确的验收标准

### 禁止的Prompt特征
- **模糊范围**："优化整个应用"、"改进所有页面"
- **无上下文**：不引用任何现有文件就要求修改代码
- **多目标混合**：同时要求UI+网络+数据库变更
- **无验收标准**：没有明确"完成"的定义

---

## 2. Prompt模板库

### 2.1 大型任务Prompt（Feature Implementation）

**适用场景**：实现一个完整的功能模块（如登录页面、数据同步、动画效果）

```
【任务类型】功能开发
【目标模块】[模块名称，如"登录"]
【输入文件】
  - PRD: [PRD/xxx.md]
  - 架构: [ARCHITECTURE/xxx.md]（如有）
  - 设计稿: [ASSETS/figma/xxx.md]（如有）

【具体要求】
1. 创建 [具体文件路径和文件名]
2. 实现以下功能点：
   - [功能点1]
   - [功能点2]
3. 遵循以下规范：
   - 架构模式：[MVVM/Clean Architecture等]
   - 代码风格：[引用AGENTS.md相关章节]
   - Apple HIG要求：[具体条款]

【验收标准】
- [ ] 功能点1可测试通过
- [ ] 功能点2可测试通过
- [ ] 符合Apple HIG设计规范
- [ ] 代码行数不超过[限制]行
- [ ] 通过静态分析检查

【约束条件】
- 不要修改与当前模块无关的文件
- 不要引入新的第三方依赖（除非PRD中已声明）
- 输出文件不超过[X]个
```

**使用示例**：
> "这是一个完整功能开发任务。目标模块是'登录页面'，PRD在 PRD/login.md，设计稿在 ASSETS/figma/login-screen.md。具体要求：1) 创建 LoginView.swift，实现邮箱密码登录UI；2) 创建 AuthViewModel.swift，处理认证逻辑；3) 遵循MVVM架构。验收标准：UI符合设计稿、错误处理完整、支持Face ID。不要修改与登录无关的文件，输出不超过5个文件。"

### 2.2 Refactor Prompt（重构）

**适用场景**：对已有代码进行重构、优化或模式改进

```
【任务类型】代码重构
【目标文件】[具体文件路径]
【当前问题】
  - [问题描述1，如"循环依赖"]
  - [问题描述2，如"可读性差"]

【重构目标】
1. 保持现有功能不变
2. 改善[具体方面：可读性/性能/可测试性]
3. 遵循[具体设计模式或架构原则]

【约束条件】
- 不改变公开API签名
- 保持所有现有测试通过
- 输出变更清单（每个文件的修改摘要）

【验收标准】
- [ ] 功能行为与重构前一致
- [ ] 通过所有现有测试
- [ ] 代码复杂度指标改善（圈复杂度降低X%）
- [ ] 重构后文件数量不超过[限制]个
```

**使用示例**：
> "这是一个代码重构任务。目标文件是 Services/AuthService.swift，当前问题是循环依赖（同时导入Network和Storage模块）。重构目标：1) 使用协议抽象解耦依赖；2) 保持公开API不变。约束条件：不改变调用方代码、保持所有测试通过、输出变更清单。"

### 2.3 Review Prompt（审查）

**适用场景**：Code Review、架构评审、设计审查

```
【任务类型】代码/架构/设计审查
【审查对象】[文件路径或模块名称]
【审查类型】
  □ Code Review    □ Architecture Review    □ Design Review

【审查重点】
1. [关注点1，如"错误处理是否完整"]
2. [关注点2，如"是否符合Apple HIG"]
3. [关注点3，如"性能瓶颈风险"]

【参考标准】
- AGENTS.md [相关章节]
- Apple HIG: [具体章节链接或描述]
- 项目架构文档: [ARCHITECTURE/xxx.md]

【输出格式】
请按以下结构输出审查结果：
1. 总体评价（1-2句话）
2. 发现的问题（按严重程度排序：🔴严重/🟡警告/💡建议）
3. 每个问题的具体位置和修改建议
4. 通过审查的确认清单
```

**使用示例**：
> "这是一个Code Review任务。审查对象是 Views/LoginView.swift 和 ViewModels/AuthViewModel.swift。审查重点：1) 错误处理是否完整（网络异常、无效输入、Token过期）；2) 是否符合Apple HIG安全指南；3) 是否有内存泄漏风险。参考AGENTS.md §13和Apple HIG Security章节。"

### 2.4 Design Prompt（设计）

**适用场景**：UI/UX设计、交互原型、Design System

```
【任务类型】UI/UX设计
【目标模块】[模块名称]
【输入参考】
  - Figma文件: [Figma链接或导出文档路径]
  - Design Tokens: [DESIGN/tokens.md]（如有）
  - Apple HIG章节: [相关章节]

【设计要求】
1. 页面/组件名称：[具体名称]
2. 包含以下元素：
   - [元素1，如"导航栏"]
   - [元素2，如"主内容区"]
   - [元素3，如"底部操作栏"]
3. 交互行为：[描述用户操作流程]

【设计规范】
- 遵循Apple HIG: [具体章节]
- 使用项目Design Tokens: [引用相关文件]
- 支持深色模式：[是/否，如支持请说明适配方式]
- 支持动态类型：[字体大小自适应要求]

【输出格式】
请按以下结构输出：
1. 页面布局描述（文字版线框图）
2. 组件列表及层级关系
3. 交互流程说明
4. 适配方案（深色模式/动态类型/多语言）
5. SwiftUI代码实现（如需要）
```

**使用示例**：
> "这是一个UI设计任务。目标模块是'设置页面'，Figma设计稿在 ASSETS/figma/settings.md。设计要求：1) 包含分组列表、开关控件、退出登录按钮；2) 支持深色模式和动态类型。遵循Apple HIG Settings章节，使用项目Design Tokens。输出页面布局描述、组件列表、交互流程、SwiftUI代码实现。"

### 2.5 Debug Prompt（调试）

**适用场景**：修复特定错误或问题

```
【任务类型】Bug修复/调试
【错误信息】[完整的错误日志或崩溃堆栈]
【涉及文件】[相关代码文件路径]
【复现步骤】
1. [步骤1]
2. [步骤2]
3. ...

【已尝试的排查】
- [已做的检查1]
- [已做的检查2]

【期望结果】[描述正确的行为]
【实际结果】[描述当前的问题行为]

【约束条件】
- 只修改与当前问题相关的代码
- 不引入新的依赖或改变API签名
- 输出修复说明（根因分析+修改方案）
```

**使用示例**：
> "这是一个Bug修复任务。错误信息：'Fatal error: Unexpectedly found nil while unwrapping an Optional value'，涉及文件 Views/SettingsView.swift:42。复现步骤：1) 打开设置页面；2) 切换深色模式；3) 应用崩溃。已尝试：检查了@State变量绑定。期望结果：深色模式切换时不崩溃。实际结果：第42行UserDefaults.standard.stringForKey('theme')返回nil导致强制解包失败。约束条件：只修改SettingsView.swift，不引入新依赖，输出根因分析。"

---

### 2.6 Architecture Prompt（架构设计）

**适用场景**：项目架构决策、模块划分、技术选型

```
【任务类型】架构设计
【项目背景】[简要描述项目类型和规模]
【当前状态】
  - 已有模块：[列出已完成的模块]
  - 技术栈：[语言、框架、依赖]

【待解决问题】
1. [问题描述1]
2. [问题描述2]

【约束条件】
- 平台限制：[如"仅iOS 16+"]
- 团队规模：[如"单人开发"/"2人团队"]
- 时间限制：[如有]

【输出格式】
请按以下结构输出架构方案：
1. 推荐的架构模式及理由
2. 模块划分图（文字版）
3. 关键设计决策及其权衡
4. 文件/目录结构建议
5. 已知风险和缓解措施
```

**使用示例**：
> "这是一个架构设计任务。项目背景：Apple原生App，包含登录、首页、设置三个核心模块，单人开发。当前状态：已完成登录模块，使用SwiftUI+MVVM。待解决问题：1) 数据持久化方案选择（CoreData vs SwiftData vs 自定义）；2) 网络层架构设计。约束条件：仅支持iOS 16+，使用本地模型（oMLX）辅助开发。输出架构模式推荐、模块划分图、关键决策权衡、目录结构建议、已知风险。"

## 3. Prompt质量检查清单

在发送Prompt之前，用以下清单自检：

| 检查项 | 合格标准 | 常见问题 |
|--------|---------|---------|
| **目标明确** | 一句话能说清要做什么 | "优化一下"、"改改看" |
| **范围限定** | 指定了具体的文件/模块名 | "整个应用"、"所有页面" |
| **有上下文** | 引用了相关文件路径或代码片段 | 凭空要求修改不引用任何文件 |
| **有验收标准** | 明确定义了"完成"的标准 | "做好了告诉我"（无具体标准） |
| **有约束条件** | 说明了什么不能做 | 没有限制导致AI过度修改 |
| **单一职责** | 一个Prompt只解决一个问题 | 同时要求UI+网络+数据库变更 |

## 4. 常见Prompt反模式（禁止）

### ❌ 模糊范围
> "优化整个应用的性能"

**应改为**：
> "优化 SettingsView.swift 中列表滚动的帧率问题，当前在滚动时有掉帧现象。涉及文件：Views/SettingsView.swift 第120-250行。验收标准：滚动帧率稳定在60fps。"

### ❌ 无上下文
> "重写登录模块"

**应改为**：
> "重构 Services/AuthService.swift，当前实现存在循环依赖（同时导入Network和Storage模块）。参考文件：PRD/login.md、ARCHITECTURE/architecture-overview.md。约束条件：保持公开API不变，所有现有测试通过。"

### ❌ 多目标混合
> "做登录页面，同时处理网络请求和数据库存储"

**应改为**：
> "先完成登录页面UI（Views/LoginView.swift），网络请求和数据库存储分开处理。本次任务只涉及UI层，不包含业务逻辑实现。"

### ❌ 无验收标准
> "把设置页面做好看一点"

**应改为**：
> "改进 Views/SettingsView.swift 的视觉层次，具体：1) 分组标题使用SF Pro Semibold 17pt；2) 行高统一为44pt（Apple HIG标准）；3) 深色模式下对比度符合WCAG AA级。参考：ASSETS/figma/settings.md。"
